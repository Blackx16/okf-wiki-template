# Open Knowledge Format

An architecture standard for organizing and linking markdown files in a flat-file database format. It uses explicit YAML properties to declare file type boundaries and relational connections.

## Core Ideas
- Metadata as Code: Relationships are represented inside the file's YAML header so they are accessible to software parsers.
- Immutable Raw Directory: Preserves unaltered primary data while allowing incremental synthesis.
